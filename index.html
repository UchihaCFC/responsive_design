¿Qué es Responsive Design y por qué lo debo utilizar?

Con responsive design estaremos maquetando nuestros diseños de forma que se puedan visualizar desde cualquier dispositivo con internet en la actualidad, ya sea un ordenador de sobremesa, un netbook, una tablet (Ipad), un smartphone (Iphone) ... etc. Gracias a seguir las pautas del responsive design nuestro maquetado se adaptará sin ningún problema a todo tipo de resoluciones con una correcta visualización.

Para poder ver con mucha más claridad lo que significa que un diseño se adapte podemos descargar Windows Resizer desde el Chrome store. y hacer pruebas con la web que haremos con skeleton o antes de empezar con la web de Mejorando.la. Podremos ver que a medida que vamos cambiando de resolución con el plugin de Chrome, los divs se van autoajustando facilitando la visualización y lectura de los mismos.

Usar Frameworks para Responsive Design

Al igual que los backend's tienen frameworks para facilitarles algo más la vida a la hora de hacer su trabajo, los frontend's con el responsive design no nos podíamos quedar atrás. es por eso que disponemos de diversos frameworks que nos ayudaran en la tediosa tarea de crear un web adaptada a todos los navegadores del mercado y a la vez compatible con todos los dispositivos.

Hay ya muchos frameworks famosos como son Bootstrap , Iunit css o Get Skeleton que es el que usaremos para este ejemplo por su simplicidad para comenzar y asentar las bases de Resposive Design para más adelante poder entender con facilidad Bootstrap y que no se nos haga nada complicado.
Todo están basados en un sistema grid por columnas lo que nos ayudará a crear la estructura de nuestro diseño y organizarla de forma clara y ordenada desde un principio.

Empezando a crear nuestra web con Skeleton

Lo primero que deberemos hacer es descargar Skeleton desde su web oficial o directamente desde su repositorio de Github y guardarlo en la carpeta donde vayamos a realizar nuestro proyecto, en el respositorio la hemos llamado "ejemplo-skeleton"

Una vez descargado podremos ver la estructura del framework:

  • images
  • stylesheets (el más importante ya que lleva todo lo que necesitamos)
    • skeleton.css (clases que tendremos que utilizar)
    layout.css (media queries)
    base.css (fuentes e imagenes, prescindible si vamos a usar estilos propios)
  • index.html (un archivo de ejemplo de skeleton)

Creando la estructura de nuestra web

Ahora que ya tenemos skeleton podemos comenzar a crear nuestro primer proyecto con él, para eso crearemos un nuevo archivo HTML llamado "ejemplo_web_skeleton.html" y continuaremos creando la estructura básica de una web HTML5 : 

    doctype html
html lang="es"
head
  meta charset="UTF-8"

Como se puede observar hemos utilizado las clases .container (para el div de todo contenido de la web incluido el header), "two-thirds column" para el ancho de dos columnas del contenido principal y "one-third column" para la columna derecha como para cada una de las columnas de nuestro contenedor de tres columnas inferior. Para el header como el nav o el footer como queremos que nos ocupen todo el ancho de la web hemos utilizado ""sixteen columns" que es la máxima medida que podemos utilizar en Skeleton.

Los estilos

Creamos un nuevo archivo css llamado estilos.css, lo guardamos en la carpeta stylesheets de nuestro proyecto y lo agregamos a nuestro archivo ejemplo_web_skeleton.html al igual que hemos hecho con los archivos css de skeleton.

En el empezaremos a dar estilos a cada una de las secciones que hemos creado y dado toque responsive para que quede algo más bonito y curioso ademas de diferenciarse unas de otras. En el repositorio que os podéis descargar desde git tenéis el archivo de ejemplo de estilos.css que a continuación veís. Es un archivo de ejemplo con que podéis jugar todo lo que queráis para practicar y llegar al diseño que más os guste.De hecho os lo recomiendo, ya que a base de hacer pruebas y practicar desde archivos prácticos es como más se aprende.
